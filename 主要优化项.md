# 主要优化项
入参校验
指针判空
拷贝代价大的参数使用 const  const& 等
拷贝代价小的参数不使用 const& 
case后加大括号
数组越界排查
智能指针误用
容器在使用前判空
new之后需要判空
new指针加 no throw 
int -> int32_t
无符号数无需判断 <0 
未使用的参数
宏定义的参数中加括号
delete空指针
内存释放，内存泄漏
napi 函数判返回值
文件关闭之后 = -1
浮点数不使用 == 判断


# 主要检出项
1. 智能指针使用不必要的引用
2. napi接口调用都没有判返回值
3. new 未使用  std::nothrow 修饰
4. 内存申请前判size，内存申请后使用前判空
